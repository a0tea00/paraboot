I started programming in Python the first time about 6 months ago, because my professor used it as the primary programming language for his course. So, I started looking into Python, and unsurprisingly, I came across Django. Yap, Django is just too big of a target floating on the ocean of Python. Of course, there are many great libraries and resources in the Python world, and I will talk about some of them later. Anyways, I decided to make something with Django now (ah... by now I mean like 6 months later...), because our team dropped the web server side part from our course project, where would've built using Django. So I decided to build a new blog for myself.

Here is my history of blogging....(really?!)

My first blog is on Blogger. It was a nice platform, but you can just use it, not "play" with it. And guess what, my next blog was Wordpress! I hosted it on Amazon EC2, and I had fun with it. But still, I think it didn't offer me too much of a playground, not that WP doesn't have customization. I want to build something from scratch. So they ended up with me deleting the blog.

What about Django? You don't need to build things from scratch...

True. I don't have to do everything by myself, but Django offers me flexibility and solid building blocks. It's like LEGO. It gives you chance to think about everything from scratch but spare you from all the tedious detail of boring things (mostly). That's what a framework is about.

 

The Stack

So I have the thing simple thing all planned out (sort of)...

http server:
ubuntu server + Apache with mode+wsgi. Because my experience from my WordPress blog. But I would love to try Nginx next.
Python environment:
Everything on top of virtualenv! It is just a time and life saver of developing python apps.
Database:
MySQL. Well.. I have been working with mysql from my whole life (little sad, did it?). It's a good database, and gets the job done. But, I am gonna try PostgreSQL and mongoDB in the near future.
Static storage:
Boto + AWS S3. Move everything to the CLOUD (yay!). So as you might have already figured, my blog is running on the cheapest EC2 instance, even using Apache to serve statics is a luxury. One day, when it grows, I will still scale to on cheap instances, because I have the CLOUD (yay!). And it works well with sorl thumbnail and django storages
Front end:
Bootstrap (with Sass ). I enjoyed the concept of a front end framework, and I really start to like things like Sass too. They keep things going smoothly (most of the time).

All this tools and software and libs are petty cool stuff, and they are OPEN source too.

So What is Next?

Well, I spent the first 5 months for getting information of the stuff above, and I spent my first month of summer intensively coding and putting things together. Finally, deployed the blog to a TEST server then to the EC2. It is not something like a million bucks project, though I had many "First time" in this journey. I learned a lot, and it makes me think about more different ways to put things together, especially on using frameworks and libs. For example, bootstrap is cool but it's large (css is 80kb +), while Pure is less than 10 kb. Django template outputing html pages is cool, but Angular.js has some "superpower" in the client end.

"It's NOT choosing what is the best, but what is the best for your project"

Now I have a bunch of new ideas, and I want to try a lot of them. So now that I have a new blog (a lot of things need to be fixed here tho - -.zzz), I am going to share my adventures here! They are not necessarily the best practices. But I value the exchange of experience, as I can learn something from them.
